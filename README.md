# p0-w1-d4-t3 · Одеть, не сломав (синтез недели)

> Тип: синтез (дни 1–4) · Репозиторий: один отдельный

## Контекст

Всю неделю ты строила «голые», но правильные страницы: семантика, формы, доступность. Теперь
первый раз одеваешь их в CSS — и главное правило: стили не должны сломать ни семантику, ни
доступность. Красивая, но недоступная страница — это шаг назад.

```
разметка д1–д3 (готовая, доступная)
          │  + style.css  (день 4)
          ▼
┌──────────────────────────────────────────────────┐
│ селекторы → находишь блоки                       │
│ box model → отступы, рамки карточек              │
│ display   → раскладка                            │
│ :focus    → фокус виден (доступность сохранена!) │
└──────────────────────────────────────────────────┘
семантика и a11y не сломаны, только одеты
```

## Что нужно сделать

Взять одну из своих готовых доступных страниц прошлых дней (форму-анкету из d3-t3 или мини-сайт
из d3-t5) и стилизовать её через внешний CSS, применяя инструменты дня 4: селекторы, box model,
`display`. Семантика разметки и доступность остаются нетронутыми, а состояния фокуса — наоборот,
подчёркиваются.

## Релизы

### Release 1 — базовая стилизация
Подключить `style.css`. Привести в порядок типографику и отступы через box model. Разметку HTML
не трогать — работать только стилями. Закоммитить до/после.

### Release 2 — раскладка и состояния
Через `display` и box model разложить блоки/карточки/поля формы аккуратно. Обязательно
стилизовать `:focus` / `:focus-visible` так, чтобы фокус был хорошо виден (доступность дня 3 не
просто сохранена, а усилена). `:hover` на интерактивных элементах.

### Release 3 — проверка, что ничего не сломалось
Прогнать страницу заново: пройти Tab’ом (фокус виден на каждом шаге), проверить скринридером
(порядок чтения не изменился), прогнать W3C-валидатор. В `CSS.md` записать, какие стилевые
решения могли бы навредить доступности и как ты этого избежала (например, не убрала `outline`
без замены, не спрятала контент через `display:none` там, где он нужен скринридеру).

## Критерии приёмки

* HTML-разметка осталась семантической и валидной (ничего не переписано ради стилей).
* Страница проходится с клавиатуры, фокус виден на каждом интерактивном элементе.
* Порядок чтения скринридером не изменился после стилизации.
* В `CSS.md` разобрано, какие CSS-приёмы опасны для доступности и как ты их обошла.
